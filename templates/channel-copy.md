# Channel Copy

Use these templates as drafts. Rewrite for the actual product and channel norms before posting.

**How to use this file.** Each title and each post sits in its own copy block — use the block's copy button and paste. Text outside the blocks is guidance, not for posting. Split-out fields (title vs body, subject vs body) map to the separate fields each platform gives you.

## Product Hunt

### Tagline

```text
[Short factual outcome, under the current Product Hunt limit]
```

### Description

```text
[Product] helps [specific user] [specific outcome] without [painful old way]. It is built for [use case] and currently supports [key capabilities].
```

### First Comment

```text
Hey Product Hunt,

I'm [name], the maker of [product].

[Product] helps [specific user] [specific outcome] without [painful old way].

I built it because [short founder story or observed pain]. The current version can:

- [Capability 1 tied to outcome]
- [Capability 2 tied to outcome]
- [Capability 3 tied to outcome]

It is best for [ideal user/use case]. It is probably not right for [honest limitation].

For launch, I'm especially looking for feedback on [specific question]. If you try it, I would love to hear where the flow is confusing, what you expected it to do, and what would make it useful enough to keep using.

Thanks for taking a look.
```

## Show HN

### Title

```text
Show HN: [Product] - [neutral factual tagline]
```

### Post

```text
Hi HN, I'm [name]. I built [product], a [plain category] that helps [specific user] [outcome].

The problem: [describe the concrete pain without sales language].

I ran into this while [personal backstory]. Existing options were [tradeoff], so I tried [your approach].

How it works: [technical or product details HN readers can inspect].

What's different: [specific difference, not superlative].

You can try it here: [link]. [Mention no-signup/demo/free access if true.]

Pricing/access: [be transparent].

I'd love feedback on [specific technical/product question], and I'm happy to answer questions about the implementation, tradeoffs, or roadmap.
```

## Reddit Or Community

### Title

```text
[Specific, non-clickbait title that says plainly what it is]
```

### Body

```text
Hi all, I'm [name], the person who built this. I checked the rules and I think this is relevant because [why it fits the community].

The problem I kept seeing was [pain]. [Product] tries to solve it by [short mechanism].

Here is the demo/link: [link]

What might be useful even if you do not use it: [lesson, benchmark, template, code, write-up, or comparison].

I'm looking for feedback on [specific question]. Happy to answer anything about how it works or why I made the tradeoffs I did.
```
